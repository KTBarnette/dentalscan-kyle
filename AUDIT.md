When I see “Not for medical use,” I wonder where this live demo is stored. Again, under the info section, a disclaimer or call-to-action explaining my privacy might be nice. A description of how the app works and what technologies the camera uses would be helpful, and instructions on how to use the app need to be clearer. Since it is a demo, the results page should have annotations describing what the real app would detect.
The camera has great object definition, but it needs to have annotations. When scanning, it might help if the camera made noise when counting down before a photo. As it stands, it might not be easy to understand for a normal person. In a real production scan, more pictures would probably make the scan more accurate.
From a technical standpoint, edge cases might want to handle flash or tell users to turn flash off. There is no mention of whether it is better to use mobile or desktop, and it is unclear how the camera handles bad lighting. It also raises questions about whether the object detection is trained to handle pictures that are not of teeth and how the disorders are being analyzed or detected.
Improvements could include creating a tutorial that shows exactly how to use the application and prompting the user to try again if the teeth are not found. It would also help to explain to the user that the results of the demo are totally private. Additionally, the camera interface should explain how to correct positioning, such as moving further left or right, and it should always be clear that if there is a medical emergency, the user should call 911 immediately.
